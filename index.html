这 12 条可以分成三层：基础控制、协作交付、系统扩展。学会它们后，你会知道什么时候让 Codex 读代码、什么时候让它改代码、什么时候让它在云端替你开 PR。

01

SAFETY

沙箱

理解 Codex 能读写哪里、不能越过哪里。把实验范围限制在工作区里，是所有自动化的安全底座。

02

PERMISSION

权限三档

低风险命令自动执行，高风险动作再确认。重点不是放开权限，而是让审查点出现在正确位置。

03

CONTROL

Steer 引导

任务跑偏时及时打断、纠偏、追加约束。越早 steering，越少返工，尤其适合 UI 和复杂改动。

04

PLAN

Plan 模式

复杂任务先让 Codex 拆步骤、标风险、列验证方式。你确认路线后再执行，避免一上来乱改。

05

PARALLEL

Git Worktree

同一个仓库开多个独立工作区，让不同任务并行推进。适合 A/B 方案、并行修 bug、隔离实验。

06

ROLLBACK

Git 一键回滚

不满意就用 Git 恢复代码状态，而不是让 Agent 猜着改回去。版本控制是你最后的刹车系统。

07

CLOUD

云端运行

把仓库授权给 Codex Cloud，让它在云端读项目、改文件、跑检查、开 PR。电脑不用一直开着。

08

MEMORY

双层记忆系统

全局记忆写个人偏好和安全规则，项目记忆写当前 repo 的约定。让 Codex 每次都从同一套规则出发。

09

PLUGIN

Plugin 三件套

连接 GitHub、Gmail、Figma 等外部工具。Codex 不只写代码，也能读取上下文并推动工作流。

10

AUTOMATION

自动化

把重复任务变成定时执行：日报、仓库巡检、邮件摘要、看板整理。让 Agent 周期性替你盯住事情。

11

SKILL

Skill

把稳定流程封装成 SOP。比如代码审查、PPT 生成、文档转换，下次直接调用，不用重新教一遍。

12

MCP

MCP + Supabase

通过 MCP 连接真实服务和数据库。Codex 可以从写代码升级到操作表结构、接口和业务数据流。

/ STUDY ORDER

推荐学习顺序

  1. 01-04 基础控制

    先学沙箱、权限、Steer、Plan，掌握如何让 Agent 安全地听你指挥。

  2. 05-08 协作交付

    再练 Worktree、回滚、云端运行、记忆系统，把任务交付变成可审查的工程流程。

  3. 09-12 系统扩展

    最后接入插件、自动化、Skill、MCP，让 Codex 进入你的长期工作系统。
